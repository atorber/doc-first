# 编码规范

## 命名规范

### 变量命名
- 使用驼峰命名：`userName`
- 布尔值使用 is/has 前缀：`isActive`

### 函数命名
- 使用动词开头：`getUserInfo()`

## 代码风格

### 缩进
- 使用 2 空格缩进

### 注释
- 函数必须有 JSDoc 注释
- 复杂逻辑必须有行内注释

## 提交规范

遵循 [Conventional Commits](https://www.conventionalcommits.org/)

- `feat:` 新功能
- `fix:` Bug 修复
- `docs:` 文档更新
- `refactor:` 重构
- `test:` 测试相关
